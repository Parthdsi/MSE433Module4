# MSE 433 — Module 4: EP Lab Scheduling Predictor

This project builds ML models to predict how long AFib ablation procedures will take in an Electrophysiology (EP) Lab, and provides a scheduling app so staff can plan their days better.

## What's in here

```
MSE433_M4_combined_dataset.csv    ← the raw dataset (150 cases)

model/
  train_model1.py                 ← trains Model 1 (current hospital data only)
  train_model2.py                 ← trains Model 2 (adds simulated app-logged features)
  ep_model1.pkl                   ← saved Model 1 (generated by train_model1.py)
  ep_model2.pkl                   ← saved Model 2 (generated by train_model2.py)
  model1_performance.png          ← predicted vs actual scatter plot
  model2_performance.png          ← predicted vs actual scatter plot
  eda_plots.py                    ← generates general EDA plots
  scheduling_eda.py               ← generates scheduling-focused EDA plots
  eda_plots/                      ← all generated plots live here

api/
  app.py                          ← Flask API that serves both models

src/
  index.html                      ← Scheduling Predictor UI (open in browser)
  LoggingApp.html                 ← Case Logging App UI (open in browser)
  SchedulingPredictor.jsx         ← React component source (for reference)
```

## How to run everything

### Prerequisites

You need Python 3 with these packages:

```
pandas numpy scikit-learn matplotlib joblib flask flask-cors
```

Install them with:

```bash
pip install pandas numpy scikit-learn matplotlib joblib flask flask-cors
```

### Step 1: Train the models

You only need to do this once (the `.pkl` files are already included, but if you want to regenerate them):

```bash
cd model
python3 train_model1.py
python3 train_model2.py
```

This will print out all the metrics and save the `.pkl` model files and performance plots.

### Step 2: Start the API

```bash
cd api
python3 app.py
```

This starts a Flask server on `http://localhost:5001`. Keep this terminal open — both apps need it running.

If port 5001 is taken, you can change it at the bottom of `app.py`.

### Step 3: Open the apps

Just open these files in your browser (double-click or drag into Chrome/Safari):

- **Scheduling Predictor:** `src/index.html`
- **Case Logger:** `src/LoggingApp.html`

Both load React and Tailwind from CDN so there's nothing to install on the frontend side.

### Step 4 (optional): Regenerate EDA plots

```bash
cd model
python3 eda_plots.py
python3 scheduling_eda.py
```

Plots get saved to `model/eda_plots/`.

## Quick overview of the two models

**Model 1** uses only data the hospital already has — which physician, what position the case is in the day, and how complex the procedure is. Best model was Linear Regression with an MAE of about 12 minutes.

**Model 2** adds pre-case flags that would come from our logging app (obesity, sleep apnea, fasting status, etc.). These are currently simulated/synthetic, so Model 2 doesn't beat Model 1 yet. The idea is that once the logging app collects real data for 30+ days, Model 2 gets retrained and should improve.

## Notes

- The API must be running for the web apps to work — they call `localhost:5001` for predictions
- The `.pkl` files are already included so you don't have to retrain unless you want to
- Logged cases from the logging app get saved to `model/logged_cases.csv`
